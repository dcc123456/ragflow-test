## 测试配置

**Agent**: 仅使用 `web-test-agent` 执行测试

**截图路径**: `/screenshots`

**日志路径**: `/test-results`

**报告路径**: `/test-reports`

**禁止**在其他路径中保存截图，日志，报告等文件

---

## 测试前准备

### 1. 查询测试任务

- 查询 `/task` 目录下的 `.md` 文件作为测试任务列表
- 示例：`billing-test.md` → 任务名 `billing`
- 按文件名排序展示，便于用户选择

### 2. 确认测试参数（逐项确认）

展示可用选项供用户勾选：

```
【测试任务】（可多选）
□ billing  (billing-test.md)
□ login    (login-test.md)
□ add-llm  (test-add-llm.md)

【测试URL】（单选 + 输入备选）
○ http://192.168.1.23:9222/
○ http://localhost:9222/
○ 其他: [___________]

【执行方式】
○ 串行执行（按顺序一个个执行）
○ 并行执行（多个任务同时执行，需要浏览器多实例）
```

### 3. 检查MCP服务

| 服务            | 检查命令                      | 成功标志                   |
| --------------- | ----------------------------- | -------------------------- |
| Playwright      | `ListMcpResourcesTool`        | 返回 `playwright` 相关资源 |
| chrome-devtools | `chrome-devtools__list_pages` | 返回页面列表               |

**异常处理**：

- 如果没有安装 → 参考 `doc\mcp-web-testing-setup\SKILL.md` 安装
- 如果未启动 → 尝试启动
- 如果启动失败 → 终止测试并通知

### 4. 测试依赖分析

分析任务间依赖关系：

| 任务    | 依赖任务 | 说明       |
| ------- | -------- | ---------- |
| add-llm | login    | 需要先登录 |
| billing | login    | 需要先登录 |

**处理策略**：

- 如果选择了有依赖的任务，自动选中其依赖任务
- 弹出提示告知用户："add-llm 依赖 login，已自动选中"

### 5. 通知测试开始

使用 `feishu-webhook` 发送：

```
【测试开始】
任务：billing, login
URL：http://192.168.1.23:9222/
执行方式：串行
开始时间：2026-04-20 10:30:00
```

### 6. 读取配置文件

- 读取 `config.md` 文件中的配置，为后续步骤使用

## 测试数据管理

### 数据准备

- 某些测试需要预置数据（如上传文件）
- 查看测试用例中是否有 `前置数据` 说明

## 通知规则（feishu-webhook）

| 场景         | 触发时机       | 消息格式                                                                            |
| ------------ | -------------- | ----------------------------------------------------------------------------------- |
| **开始**     | 任务启动时     | `【测试开始】\n任务：{任务名}\nURL：{URL}\n执行方式：{方式}\n开始时间：{时间}`      |
| **进度**     | 每5分钟        | `【进度更新】\n任务：{任务名}\n当前：{当前步骤}\n进度：{百分比}%\n预计剩余：{时间}` |
| **步骤完成** | 每步执行完成   | `【步骤完成】\n任务：{任务名}\n步骤：{步骤名}\n结果：{结果}`                        |
| **完成**     | 任务正常结束   | `【测试完成】\n任务：{任务名}\n结果：✅通过/❌失败\n耗时：{时间}\n详情：{关键结果}` |
| **错误**     | 任务异常时     | `【测试错误】\n任务：{任务名}\n错误：{错误信息}\n步骤：{失败步骤}\n堆栈：{摘要}`    |
| **全部完成** | 所有任务结束时 | `【全部完成】\n总计：{N}个任务\n通过：{M}个\n失败：{K}个\n耗时：{总耗时}`           |
